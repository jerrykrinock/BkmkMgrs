\subsection{Detailed Description}
A managed object which stores the configuration settings of a \hyperlink{class_bmx_bk}{BmxBk}. 

One, and only one of these, is inserted into a new \hyperlink{class_bmx_bk}{BmxBk} when it is initially created.

There are two ways to handle the Automatic configuration (configureAutomatically). \begin{itemize}
\item The first way would be to to write custom getters for all of the {\em collection-support\/} attributes: \begin{itemize}
\item satchAll1 \item satchAll2 \item hasBar \item hasMenu \item hasOhared \item hasUnsorted \item rootLeavesOk \item rootSoftainersOk \item rootNotchesOk \item softiesAmongFixturesOk \end{itemize}
Each custom getter would have to iterate through the internalizers and calculate the required setting.~ And this would be calculated during normal program operation. \item The second way would be to set KVO observers which, if configureAutomatically is YES, iterate through the above list of affected attributes and calculate a required setting for each one and set it.~ This will be done only when internalizers were changed, practically, once a year.~ Two observers are required, though: One to observe 'browfile' of \hyperlink{interface_ixternalizer}{Ixternalizer} and one to observe 'internalizers' of \hyperlink{interface_bmx_bk_config}{BmxBkConfig}. 

Obviously, the second way is less code and more efficient to run, so I do that.~ It will also cause the disabled attribute settings to display as the current, automatically-calculated values instead of holding on to the last manually-set values.~ I feel that displaying this more correct information to users would be better than holding onto the last manually-set values, since it only takes nine clicks at most to restore the last manually-set values. \end{itemize}
