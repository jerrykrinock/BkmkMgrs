\subsection{Detailed Description}
A class for fetching Browfiles from and inserting Browfiles into the application-wide in-memory managed object context.~ To avoid duplication, a \hyperlink{interface_browfile}{Browfile} must be obtained using one of these class methods. 

The only primary reason why Browfiles are managed objects is so that they can be copied and stored in the object graph of the \hyperlink{class_bookshelf}{Bookshelf} documents.

I suppose that, since Browfiles could be re-used, they \char`\"{}central\char`\"{} store of Browfiles could be in application's shared sqlite moc instead of the in-memory moc.~ But browfiles don't take too much effort to create, and since they can change, I decided to fetch and re-use them while the app is running (see +browfileWithParts:), but throw them away when the app quits.

Returns an array of all available browfiles of extoreMedia BkMxExtoreMediaThisUser for the current Mac user, inserted into the app-wide shared in-memory managed object context provided by SSYMOCManager. 

The array consists of the results from +\mbox{[}self allBrowfilesForLocalAppsThisUser\mbox{]} followed by results from +\mbox{[}self allBrowfilesForWebAppsThisUser\mbox{]}, and for each of the latter, an item with a nil profile which could be used to create a new profile.

The result can be used to populate the popup menus of external sources and destinations.

\begin{Desc}
\item[Returns:]The array of browfiles, or an empty array if no qualified browfiles were found. \end{Desc}
