\subsection{Detailed Description}
A Quit Watcher.~ Will watch quatches (quit watches) and, when an app specified by a quatch quits, will launch a worker tool with the argument specified in the quatch. 

Note that, because the quatches and workerPath are stored as instance variables, and there are no setters, it is necessary to destroy this object and create a new one when the underlying values in the user defaults change.~ In practice, this is done by killing and then restarting the Quatch tool in which this thing lives.~

This would not be necessary if this class' -handleAppQuit method would read the user defaults whenever needed.~ However, that might cause this agent to use more CPU time whenever an app quits, and since it was easy to do the instance variables, and natural to quit and restart the Quatch tool in -\mbox{[}\hyperlink{class_bookshelf}{Bookshelf} syncLaunchdAgents\mbox{]} anyhow, and since instance variables was the method I used in Bookwatchdog, I decided to use instance variables.~
